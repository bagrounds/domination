# Re-theme game to avoid intellectual property concerns

**Source:** New initiative

**Status:** Open

## Description

The current game mechanics are inspired by deck-building card games. To ensure we aren't using any existing intellectual property, we should re-theme the game with original concepts while keeping the mechanics identical.

## Current → Proposed Theme Mapping

### Core Concepts

| Current | Proposed | Flavor |
|---------|----------|--------|
| Buys | Conjurations | "How many spells you can invoke this turn" |
| Money/Treasure | Mana/Essence | "Magical energy used to conjure new spells" |
| Actions | Casts | "How many spells you can cast this turn" |
| Cards | Runes / Glyphs | "Magical inscriptions of power" |
| Deck | Grimoire | "Your personal book of spells" |
| Hand | Focus | "The runes you're currently channeling" |
| Discard pile | Drift | "Spent runes returning to your grimoire" |
| Trash | Void | "Runes banished to the void" |
| Supply | Arcanum | "The shared library of available runes" |
| Victory points | Dominion / Sovereignty | "Your claim over the realm" |
| Game | Rite / Ritual | "A contest of arcane mastery" |
| Player | Arcanist | "A practitioner of the arcane arts" |
| Turn | Cycle | "One rotation of magical energy" |

## Game Title Ideas

- **Arcanum** — "A contest of arcane mastery"
- **Grimoire** — "Build your book of power"
- **Sovereignty** — "Claim dominion through arcane arts"
- **Runeweaver** — "Weave runes of power"
- **The Rite** — "An arcane ritual of power"

## Implementation Notes

- All mechanical changes are purely cosmetic (rename variables, strings, UI text)
- No game logic changes required
- Card effects, costs, and values remain identical
- The re-theme can be done incrementally, starting with UI text and card names
- Consider making the theme configurable so both themes can coexist
